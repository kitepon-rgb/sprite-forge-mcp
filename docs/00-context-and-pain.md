# 00 — Context and Pain（なぜ sprite-forge を作るのか）

> **状態（2026-06-18）**: ここで定義した全痛点は、ツールの瘢痕ルールとして実装済み（`CLAUDE.md`「必守の瘢痕ルール」）。本書は設計判断の根拠＝歴史的に有効。

> このドキュメントは rpgdev のアセット制作で実際に起きた苦闘の記録（Codex/Claude セッションログ採掘・メモリ・rpgdev CLAUDE.md の「画像生成 最重要ルール」）を蒸留したもの。**ツールが解くべき問題の定義**であり、設計判断の根拠。

## 苦闘の規模（事実）

- `rpgdev/public/assets/sprites/` は約 **520 ファイル**中、採用は実質十数枚。残りは候補・デバッグ・比較画像の墓場。
- あるダメージ版1枚だけで **v1 → v39**（中間比較込みで ~196 ファイル）。別の版は各部位の微修正の山、別の版は v2〜v7c＋右腕除去マスク v1–v5 連発、別の版は purple keying。
- 経路は **その場のクラウド生成＋手作業 Pillow**＝再現性が無く、同じ失敗を何度も踏んだ。

## 役割分担（当時）

- 生成 = 対話型 Codex（このプロジェクトで何度も生成し系統が揃う）。
- 機械処理 = Claude（PIL の透過・bbox/高さ合わせ・posterize・比較グリッド）。
- 画像生成自体は外部の生成ツールに任せ、Claude は機械処理（透過・bbox・posterize・比較）に徹する。

## 6つの痛点（ツールが構造的に殺すべき対象）

1. **クロマキー透過の黒漏れ**。単純な外周 flood-fill では手足/リボンの隙間の「閉じた黒」が残る。**ある採用版に黒が20万px以上**残存。→ 監査（`alpha>24 & r,g,b<28` の画素数）が必須化。
2. **ベースとダメージ版で pose/canvas/bbox がズレ、CSS 配置が崩れる**（最難関）。表示・輪郭が乱れ、元画像から1ドットもずらせない。
3. **美麗アニメへの画風ドリフト**。`retro RPG / pixel art / old JRPG sprite / limited palette / chunky pixel clusters` を抜くと崩れる＝この必須語句は**必須・非交渉**（省略不可）。
4. **手作業加工は全滅＝永久禁止**。布/肌/輪郭の描き足し・合成・救済は成功例ゼロ。直すなら**再生成 or 機械的処理**のみ。
5. **キャラ同一性・画風統一の維持**（既存キャラ群で画風を揃え、同一キャラを変種間で保つ）。
6. **「正解の見た目」の主観性で版が爆発**。結果のパス探し・インライン表示・ゲーム表示サイズでの確認の煩わしさ。

## 個別の教訓（再発防止）

- **元画像の取り違え事故**：`faithful`/`vh700_tight` という派生クロップを「元画像」と誤認し、振り上げた右手を切り落とした版を量産。→ 正典は**ユーザー添付の全身（例 1536×1024）のみ**。派生を元と扱わない。
- **黒背景は危険**：透過前提のクロマキーに黒を使うと漏れる。安全色（`#ff00cc` 等、キャラ色と衝突しない珍色）を使う。
- **手指/腕ポーズの literal 指定で生成器が破綻**（胸を手で覆う等の直接指定で崩壊）→ 婉曲表現で回避。
- **採用判断は view での主観**だった（v39「stern」を v37/v38 より採用）。39版という数字が主観性を物語る。

## ツールが満たすべき要件（痛点 → 機能）

| 痛点 | sprite-forge が構造的にどう殺すか |
|---|---|
| #1 黒漏れ | LayerDiffuse で **RGBA ネイティブ生成**（クロマキー不要）。残存経路は安全クロマ既定＋黒漏れ監査ゲート |
| #2 pose/canvas ズレ | Qwen-Image-Edit の **masked 編集（領域外は不変）**＋ControlNet pose ロック＋採用ゲートで bbox/canvas 差を機械検証 |
| #3 画風ドリフト | 必須語句の**自動注入**＋画風 LoRA（既存キャラ群で学習） |
| #4 手作業禁止 | UI/ツールは**指し示す（マスク/点/線）だけ**。修正は再生成/inpaint。手描きペイント機能は持たない |
| #5 同一性/統一 | キャラ LoRA＋画風 LoRA、SAM2 点マスク、元画像を base にした編集 |
| #6 主観の版爆発 | **複数案バッチ生成→ギャラリー並べ比較→採用**を1画面で。seed 固定で再現。パス探し不要 |
